🔄 完整流程拆解

1 飞书 → OpenClaw 接收指令

OpenClaw 飞书通道

贺佬湿在飞书里直接发消息：帮我开发个贪吃蛇游戏，推送到 GitHub，部署到 Vercel 和 Cloudflare。

飞书通过 WebSocket 连接把消息实时路由给本地 Mac 上的 OpenClaw 助手 子书，无需任何中转服务器。

💡 关键点：OpenClaw 自带飞书通道插件，配好 App ID/Secret 就能打通。消息通过 WebSocket 实时推送到本地 Gateway，延迟极低——你在飞书里聊天，Agent 在你自己的电脑上干活。

2 OpenClaw 理解意图 & 准备环境

模型推理 环境准备

子书（DeepSeek V4 Pro 驱动）解析需求，自动制定执行计划：

  1. 创建 myclaudecode 工作目录
  2. 启动 Claude Code 编写代码
  3. GitHub 仓库创建 & 推送
  4. 多平台部署（Vercel + Cloudflare）
mkdir -p ~/.openclaw/workspace/myclaudecode    # 子书执行

3 Claude Code 生成游戏代码

Claude Code PTY 代理交互

子书通过 PTY（伪终端）启动 Claude Code，传入编程需求：

cd ~/.openclaw/workspace/myclaudecode && claude    # 启动
→ 帮我开发一个完整的贪吃蛇游戏……                    # 需求

Claude Code 接到指令后：

  • 🧠 分析需求：纯 HTML/CSS/JS 单文件、Canvas 渲染、键盘+触屏操控
  • ⏳ Synthesizing（思考生成）约 1 分钟
  • ✍️ 直接写入 snake.html — 475 行完整代码
  • ✅ 子书代为确认权限，文件写入磁盘
💡 关键点：OpenClaw 通过 PTY 操控 Claude Code，不仅能发指令、读输出，还能代它确认权限弹窗。两个 Agent 不需要人类在中间当「传话筒」。

4 Git 提交 & 推送 GitHub

GitHub 子书编排

Claude Code 写好代码后执行 git commit，但发现 gh CLI 未安装。子书自动接管：

# 子书无缝接管环境安装
brew install gh                                  # 安装 GitHub CLI
echo "ghp_xxx" | gh auth login --with-token     # Token 认证

# 创建仓库 + 推送
gh repo create hemin1003/snake-game --public --push

# 复制 index.html 启用 GitHub Pages
cp myclaudecode/snake.html index.html
git add . && git commit && git push
💡 关键点：这是「双 Agent 协作」的最佳体现。Claude Code 专注于代码生成，遇到环境依赖缺口时，OpenClaw 自动介入解决，不让流程中断。

5 三平台自动部署

GitHub Pages Vercel Cloudflare

代码推送到 GitHub 后，子书依次完成三个平台的部署：

🐙 GitHub Pages

代码推送后自动触发构建，自定义域名 heminit.com 直接生效。子书通过 API 确认构建状态，无需手动操作。

▲ Vercel（7 秒上线）

npx vercel deploy --token=vcp_xxx --prod --yes
→ Production: myclaudecode.vercel.app  ✅
  • 零配置、自动检测静态站点
  • 全球 CDN 分发、自动 HTTPS
  • 每次推送自动部署到生产环境

⛅ Cloudflare Pages（全球加速）

CLOUDFLARE_API_TOKEN="cfut_xxx" \
npx wrangler pages deploy . \
  --project-name=snake-game --branch=main
→ snake-game-11f.pages.dev  ✅
  • 7 个文件秒传、2.45 秒上传完成
  • 全球 330+ 节点边缘网络
  • 免费无限带宽

🔑 关键技术点

1. OpenClaw 多通道消息路由

通过插件体系支持飞书、微信、钉钉、Discord。消息通过 WebSocket 实时推送到本地 Gateway，Agent 回复自动返回原通道。

// openclaw.json 飞书通道配置
"feishu": {
  "enabled": true,
  "appId": "cli_xxx",
  "connectionMode": "websocket"
}

2. PTY 代理控制 Claude Code

OpenClaw 通过伪终端（PTY）启动并操控 Claude Code，实现：发送指令 → 读取响应 → 代确认权限 → 继续执行，全程无需人工介入。

exec("claude", { pty: true })
process.paste("开发需求...")
process.sendKeys(["enter"])       // 发送
process.sendKeys(["enter"])       // 确认权限弹窗

3. 双 Agent 协作

子书（OpenClaw）负责环境编排 + 工具安装 + 多平台部署；Claude Code 负责纯代码生成。遇到环境依赖缺口时自动切换、互不阻塞。

4. Token 驱动的全自动化

GitHub、Vercel、Cloudflare 全部通过 API Token 驱动，用户只需提供一次 Token，Agent 自动完成认证、部署、验证全链路。
